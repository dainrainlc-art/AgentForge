# P1 任务实施计划

## 为什么

根据架构改进计划，P0 任务已全部完成。现在需要完成剩余的 P1 任务，以提升系统的测试覆盖率、API 文档完整性和 AI 审核工作流功能。

## 变更内容

### 新增功能

- **完整测试体系**: 为核心模块和业务引擎编写单元测试，编写集成测试
- **API 文档自动化**: 集成文档生成器，自动生成 OpenAPI 文档和 SDK
- **AI 审核工作流增强**: 完善审核 UI、历史追溯、驳回分析功能

### **BREAKING** 变更

- 无（所有变更均为新增或增强）

## 影响范围

### 受影响的能力

- 测试覆盖率从 35% 提升到 70%+
- API 文档完整性和可访问性
- AI 审核工作流的用户体验和功能性

### 受影响的代码

- `tests/unit/` - 新增单元测试文件
- `tests/integration/` - 新增集成测试文件
- `integrations/api/docs_generator.py` - 集成到主应用
- `frontend/src/pages/AuditQueue.tsx` - 增强审核界面
- `integrations/api/audit_api.py` - 新增审核历史和分析接口

## 新增需求

### 需求：测试体系

系统 SHALL 具备完整的测试覆盖：

#### 场景：核心模块单元测试
- **WHEN** 运行 `pytest tests/unit/test_core.py`
- **THEN** Agent、MemoryStore、ModelRouter 等核心模块测试通过，覆盖率 ≥ 70%

#### 场景：业务引擎单元测试
- **WHEN** 运行 `pytest tests/unit/test_engines.py`
- **THEN** Fiverr、Social、Knowledge 等业务引擎测试通过，覆盖率 ≥ 70%

#### 场景：集成测试
- **WHEN** 运行 `pytest tests/integration/`
- **THEN** 数据库、API、工作流集成测试通过，覆盖主要业务流程

### 需求：API 文档

系统 SHALL 提供完整的 API 文档：

#### 场景：文档自动生成
- **WHEN** 应用启动
- **THEN** 自动生成 OpenAPI 文档到指定目录

#### 场景：SDK 生成
- **WHEN** 运行文档生成器
- **THEN** 生成 Python 和 TypeScript SDK

### 需求：AI 审核工作流增强

系统 SHALL 提供完整的审核工作流体验：

#### 场景：审核 UI 优化
- **WHEN** 用户访问审核页面
- **THEN** 可以看到优化的审核队列、内容对比、批量操作功能

#### 场景：审核历史追溯
- **WHEN** 用户查看审核历史
- **THEN** 可以看到完整的审核记录、版本历史、操作日志

#### 场景：驳回分析
- **WHEN** 管理员查看驳回统计
- **THEN** 可以看到驳回原因分析、趋势图表、改进建议
