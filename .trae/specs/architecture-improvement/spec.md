# AgentForge 架构报告改进计划

## 为什么

根据《AgentForge 架构规划与实际实施对比分析报告》（AFR-2026-001），识别出系统存在以下关键差距和风险：

1. **高风险**：缺少自进化机制（记忆巩固、自我检查）
2. **高风险**：缺少单元测试体系，代码质量无法保证
3. **中风险**：Tauri 桌面端功能不完整
4. **中风险**：API 文档不完整，增加集成难度
5. **中风险**：AI 审核工作流 UI 不完整
6. **低风险**：部分外部集成缺失（Telegram、飞书等）

本计划旨在系统性地解决这些问题，按照轻重缓急排序实施。

## 变更内容

### 新增功能

- **自进化机制**：实现每日记忆巩固、自我检查、任务复盘
- **完整测试体系**：单元测试、集成测试、E2E 测试
- **Tauri 桌面端增强**：AI 聊天界面、配置管理、系统托盘
- **API 文档自动生成**：集成 docs_generator，生成 OpenAPI 文档和 SDK
- **AI 审核工作流完善**：完整审核历史、驳回分析

### 修改功能

- **Fiverr 主页优化**：增强数据分析和优化建议功能
- **社交媒体分析**：完善效果追踪和数据分析
- **前端状态管理**：引入统一的状态管理方案

### **BREAKING** 变更

- 无（所有变更均为新增或增强，不破坏现有功能）

## 影响范围

### 受影响的能力

- AgentForge Core 自进化能力
- 测试覆盖率（从 0% 提升到 80%+）
- 桌面端用户体验
- API 文档完整性
- AI 审核工作流完整性

### 受影响的代码

- `agentforge/core/self_evolution.py` - 需要增强
- `tests/` - 新建测试目录和文件
- `desktop/src-tauri/` - 增强桌面端功能
- `integrations/api/docs_generator.py` - 集成到主应用
- `frontend/src/pages/AuditQueue.tsx` - 增强审核界面

## 新增需求

### 需求：自进化机制

系统 SHALL 实现以下自进化能力：

#### 场景：每日记忆巩固
- **WHEN** 每天凌晨 3 点
- **THEN** 系统自动读取 MEMORY.md，去重并提取洞察，更新 MEMORY.md

#### 场景：每日自我检查
- **WHEN** 每天凌晨 4 点
- **THEN** 系统自动分析错误日志，生成诊断报告，提出自动修复建议

#### 场景：任务复盘
- **WHEN** 任务执行完成后
- **THEN** 系统记录执行过程，提取经验教训，更新技能库

### 需求：测试体系

系统 SHALL 具备完整的测试覆盖：

#### 场景：单元测试
- **WHEN** 运行 `pytest tests/unit/`
- **THEN** 所有核心模块和业务引擎测试通过，覆盖率 ≥ 70%

#### 场景：集成测试
- **WHEN** 运行 `pytest tests/integration/`
- **THEN** 所有集成测试通过，覆盖主要业务流程

#### 场景：E2E 测试
- **WHEN** 运行 `pytest tests/e2e/`
- **THEN** 关键用户路径测试通过

### 需求：Tauri 桌面端增强

系统 SHALL 提供完整的桌面端体验：

#### 场景：AI 聊天界面
- **WHEN** 用户打开桌面应用
- **THEN** 可以看到类似 ChatGPT 的聊天界面

#### 场景：配置管理
- **WHEN** 用户需要修改配置
- **THEN** 可以通过 GUI 界面修改和保存配置

#### 场景：系统托盘
- **WHEN** 应用运行在后台
- **THEN** 显示系统托盘图标，支持快速操作

### 需求：API 文档完善

系统 SHALL 提供完整的 API 文档：

#### 场景：OpenAPI 文档
- **WHEN** 访问 `/docs`
- **THEN** 可以看到完整的 Swagger UI 文档

#### 场景：SDK 生成
- **WHEN** 运行文档生成器
- **THEN** 自动生成 Python 和 TypeScript SDK

## 修改需求

### 需求：AI 审核工作流

**原需求**：实现基本的审核队列和审批功能

**修改后**：实现完整的审核工作流，包括：
- 完整的审核历史追溯
- 驳回原因分析
- Telegram/邮件通知（可选）
- 版本对比功能

## 移除需求

无

## 优先级排序

根据架构报告中的建议，按轻重缓急排序：

### P0 - 紧急重要（1 周内完成）

1. **自进化机制实现** - 高风险，影响系统核心价值
2. **Tauri 桌面端完善** - 中风险，影响用户体验

### P1 - 重要（1 个月内完成）

3. **测试体系建立** - 高风险，影响代码质量
4. **API 文档完善** - 中风险，影响集成效率
5. **AI 审核工作流完善** - 中风险，影响功能完整性

### P2 - 次要（1 季度内完成）

6. **Fiverr 主页优化** - 中风险，影响业务价值
7. **社交媒体分析完善** - 中风险，影响数据分析
8. **外部集成补充** - 低风险，按需实现
