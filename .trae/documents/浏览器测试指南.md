# AgentForge 浏览器测试指南

## 服务状态

✅ 服务运行正常
- API地址: http://localhost:8000
- 健康状态: healthy
- LLM提供商: baidu_qianfan
- Redis: ✅ 正常
- Qdrant: ✅ 正常

- 默认模型: glm-5

## 测试步骤

### 步骤 1: 打开 Swagger UI
1. 在浏览器地址栏输入: http://localhost:8000/docs
2. 猉待页面加载完成

### 步骤 2: 测试健康检查 API
1. 找到 `/api/health` 端点
2. 点击 "Try it out"
3. 点击 "Execute"
4. 查看返回结果

**预期结果**:
```json
{
  "status": "healthy",
  "timestamp": "...",
  "version": "1.0.0",
  "services": {
    "llm": {"status": "healthy", "provider": "baidu_qianfan"},
    "memory": {"status": "healthy", "redis": true, "qdrant": true}
  }
}
```

### 步骤 3: 用户注册
1. 找到 `/api/auth/register` 端点
2. 点击 "Try it out"
3. 在请求体中输入:
   ```json
   {
    "email": "mytest@example.com",
    "password": "mypassword123",
    "name": "测试用户"
   }
   ```
4. 点击 "Execute"

**预期结果**:
```json
{
  "id": "user_X",
  "email": "mytest@example.com",
  "name": "测试用户",
  "created_at": "..."
}
```
### 步骤 4: 用户登录（获取Token）
1. 找到 `/api/auth/login` 端点
2. 点击 "Try it out"
3. 在请求体中输入:
   ```json
   {
    "email": "mytest@example.com",
    "password": "mypassword123"
   }
   ```
4. 点击 "Execute"
5. **复制返回的 `access_token` 值**

**预期结果**:
```json
{
  "access_token": "eyJ...",
  "token_type": "bearer",
  "expires_in": 86400
}
```
### 步骤 5: 配置认证
1. 点击页面右上角的 "Authorize" 按钮
2. 在弹出框中输入: `Bearer YOUR_TOKEN`
   - 替换 YOUR_TOKEN 为步骤4获取的token
3. 点击 "Authorize"

### 步骤 6: 发送聊天消息
1. 找到 `/api/chat/message` 端点
2. 点击 "Try it out"
3. 在请求体中输入:
   ```json
   {
    "message": "你好，请介绍一下你自己"
   }
   ```
4. 点击 "Execute"

**预期结果**:
```json
{
  "response": "你好！我是GLM...",
  "agent_id": "agent_user_X",
  "timestamp": "...",
  "memories_used": 0
}
```
### 步骤 7: 获取对话历史
1. 找到 `/api/chat/history` 端点
2. 点击 "Try it out"
3. 点击 "Execute"

**预期结果**:
```json
{
  "messages": [...],
  "total": 2
}
```
### 步骤 8: 获取Agent状态
1. 找到 `/api/chat/status` 端点
2. 点击 "Try it out"
3. 点击 "Execute"

**预期结果**:
```json
{
  "agent_id": "agent_user_X",
  "name": "AgentForge-user_X",
  "created_at": "...",
  "conversation_count": 2,
  "status": "active"
}
```

## 故障排除

### 问题1: Token无效
- 检查Token是否正确复制
- 磀保Token没有多余空格
- 磀保Token没有过期

### 问题2: 服务无响应
- 检查服务是否启动
- 检查端口是否正确
- 查看服务日志
### 问题3: 聊天响应慢
- 检查网络连接
- 检查LLM服务状态
- 尝试使用更快的模型
